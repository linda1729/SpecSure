# 海岸带高光谱数据分类系统 — 前端草图详细

---

## 一、整体愿景与界面风格

- **系统形态**：  
  - 可以是桌面应用（类似 Electron / Qt）或 Web 应用（SPA 单页应用）。  
  - 外观类似现代桌面软件：顶端有窗口标题栏和菜单栏，下方是主内容区。

- **布局骨架（所有界面共用）：**
  1. **最上方窗口标题栏**  
     - 文案示例：`Coastline Hypersperrical Data Classification System – Prototype v1.0`  
     - 右侧有标准窗口按钮（最小化、最大化、关闭），草图中使用系统默认样式。

  2. **窗口标题栏下方：顶端菜单栏**  
     - 菜单项按顺序：`File`, `Data`, `Model`, `Results`, `Help`  
     - 样式类似桌面软件菜单（浅灰背景，文字水平排列）。

  3. **左侧垂直导航栏（侧边栏）**  
     - 整个系统的主导航。所有页面中保持不变，只是高亮项不同。  
     - 纵向排列四个按钮，每行左侧有小图标，右侧是文本：
       1. `Data Import`
       2. `Proprecessing / Preprocessing`（图上有拼写错误，但逻辑上是“数据预处理”）
       3. `Model Training`
       4. `Results & Evaluation`
     - 当前选中的模块背景为亮蓝色（active 状态），左侧有一条浅蓝竖线或对勾图标，文字为白色。  
     - 非选中项为浅灰背景，文字为深灰色，图标线框样式。

  4. **右侧主工作区**  
     - 占据绝大部分宽度，用来显示当前模块的具体内容。
     - 典型布局：顶部是当前模块标题（例如 `Browse Import` / `Data Preprocessing` / `Model Training`），标题下是按功能分组的卡片式区域（白色卡片、带阴影、圆角）。

## 二、系统主界面 / 数据导入模块草图（对应 PDF 第 1 页）

### 2.1 整体布局概述

- 左侧：导航栏当前高亮 **Data Import**。
- 右侧：主区域顶部标题 `Browse Import`，下方分为三个纵向区域：
  1. 文件上传区域
  2. 文件列表区域
  3. 数据预览区域（图像 + 光谱曲线）

你可以理解为一个典型的“导入向导”首页。

### 2.2 标题与面包屑

- 标题行示例：`Browse Import`
- 标题上方没有额外面包屑，直接位于主内容卡片上。

### 2.3 文件上传区

放在主卡片最上方，样式为一块横向区域：

1. **按钮：`Browse Files...`**
   - 蓝色主按钮，放在左侧。
   - 按钮文本为白色，左边可以有一个“文件夹 / 上传”图标。
   - 点击行为：弹出文件选择对话框。

2. **拖拽提示文本**  
   - 按钮下方或右侧一行浅灰文字：`Drag & Drop Files Here`
   - 表明此区域支持拖拽上传。

> 可以把这部分实现为一个 DropZone 组件：点击触发 `<input type="file">`，拖进去有高亮边框。

### 2.4 文件列表区域

紧接在上传区下方，是一个简单的表格：

- 表头（水平）：
  - `Filename`
  - `Status`
  - `Size`

- 表体（示例一行）：
  - `coastline_hsi_data.tif`  
  - `Ready`  
  - `1.2 GB`

表格风格：

- 表头背景略浅，文字加粗。
- 行之间有分隔线。
- 右边可以预留更多列（如“删除”图标、“查看详情”等）。

### 2.5 数据预览区

文件列表下方是 **Data Preview** 区域，下方两个子模块：**图像预览** 和 **光谱曲线**。

1. **小标题**：`Data Preview`
2. **图像预览（左上）**  
   - 一个小矩形卡片，展示伪彩色海岸线影像：
     - 画面包含：海岸线、水体（蓝色）、沙滩 / 滩涂（浅黄色）、植被（绿色）等。
     - 目的是让用户快速确认数据内容及空间范围。
   - 图片下暂无控件，只是静态预览。后续可扩展为可拖拽/缩放的图层视图。

3. **光谱曲线图（下方占整行或图像右侧）**  
   - 坐标轴文字：`Spectral Profile (Wavelength vs. Reflectance)`  
   - 横轴：波长（例如 400–2500 nm），以数值刻度显示。  
   - 纵轴：反射率（0–20 或 0–1，根据具体数据缩放）。  
   - 曲线：草图中有多条不同颜色的折线，表示不同像元或类别的光谱曲线。  
   - 未来交互想象：
     - 在上方图像中点击一个像元时，这里的曲线会更新为该像元的光谱。
     - 支持多线叠加比较（例如多种地物类型的光谱）。

## 三、数据预处理模块草图（对应 PDF 第 3–4 页）

当左侧导航选中 **Pro(Pre)processing** 时，右侧显示“数据预处理”界面。

### 3.1 整体视觉

- 右侧主卡片顶部标题：`Data Preprocessing`
- 下方竖向依次为三个分组模块：
  1. Noise Reduction（降噪）
  2. Band Selection（波段选择）
  3. Data Normalization（数据标准化）
- 卡片背景为白色，四角有轻微圆角，外侧有阴影，整体呈现代 Dashboard 风格。

### 3.2 模块一：Noise Reduction（降噪）

位置：`Data Preprocessing` 卡片的最上方。

内容结构：

1. **模块标题行**：左侧文字 `Noise Reduction`，右侧一个总开关 Toggle（蓝色表示 ON）。
2. **过滤器类型选择**：
   - 此模块内部有一个“滤波类型标签栏”或下拉框，草图中采用的是文本按钮：
     - 例如：`Gaussian Filter` 被高亮，代表当前使用高斯滤波。
     - 也可以扩展为：`Mean Filter`、`Median Filter` 等。
3. **Filter Size 调节**：
   - 标签：`Filter Size`
   - 下面是一个滑块（Slider），右侧标出当前值，例如 `3x3`。
   - 滑块从小核（3×3）到大核（7×7 或 11×11），可视化滤波强度。

> 推荐交互：在滤波模块下方或侧边可以展示一条光谱或一小块影像的“滤波前后对比”。

### 3.3 模块二：Band Selection（波段选择）

紧接降噪模块下方，结构如下：

1. **模块标题与开关**：
   - 标题：`Band Selection`
   - 右侧一个 `Select Subset Bands` 开关（Toggle），控制是否启用“子集波段选择”。

2. **光谱范围交互图**：
   - 一张带横轴波长刻度的光谱曲线图：
     - 横轴：`Wavelength Range (µm)` 或 `nm`
     - 纵轴：`Reflectance`
     - 图中可以看到几条典型地物曲线。
   - 在曲线图上方有一个浅色矩形框（“选区”），表示当前选定的波段范围：
     - 用户可通过拖拽选区的左右边界，交互式调整波段范围。
     - 或直接从两端的文本框中输入起止波段索引 / 波长。

3. **自动波段选择（可选，在后续实现时扩展）**：
   - 在图的下方可加入一行选项：
     - 单选按钮：`PCA Top K Components`  
     - 单选按钮：`Automatic Selection (e.g. based on class separability)`  
   - 用户可以选择“手动框选”或“自动算法选择”。

### 3.4 模块三：Data Normalization（数据标准化）

位于最下方。

1. **模块标题与开关**：
   - 标题：`Data Normalization`
   - 下方有一行说明，例如：`Apply Normalization (nm)`，右侧为 Toggle 开关（蓝色表示启用）。

2. **方法选择**：
   - 下面是一个按钮组或单选框列表。例如：
     - `Min-Max Scaling`
     - `Z-score Standardization`
   - 草图中只明确写出了 `Min-Max Scaling`，你可以在实现时补充其他选项。

### 3.5 全局按钮：Apply Preprocessing

- 位于整个卡片的底部居中位置，是一枚蓝色主按钮：`Apply Preprocessing`。
- 点击后：
  - 系统执行所有当前勾选/启用的预处理操作（降噪 + 波段选择 + 标准化）。
  - 右下角可以弹出 Toast 或状态条：`Preprocessing Completed: Removed noisy bands, applied Gaussian filter (3x3), Min-Max normalized.`

## 四、分类模型训练模块草图（对应 PDF 第 4–5 页）

当左侧导航选中 **Model Training** 时，右侧显示“模型训练”界面。

### 4.1 整体布局

- 主标题：`Model Training`
- 副标题/小标题：`Select & Configure Models`
- 下方主要由两个“模型卡片 + 数据划分区域 + 训练按钮”构成。

为了突出“模型对比”，页面中心是两个并排的卡片：

- 左：Machine Learning Model
- 右：Deep Learning Model

### 4.2 模型卡片一：Machine Learning Model（机器学习模型）

位置：左侧卡片。

组成：

1. **卡片标题与开关**  
   - 标题：`Machine Learning Model`
   - 右上角一个开关标记 `ON`（Toggle），允许启用/禁用该模型。

2. **算法选择**  
   - 行首标签：`Algorithm`
   - 右侧下拉框（Dropdown），默认值为 `Support Vector Machine (SVM)`。
   - 下拉菜单中可以包括：`SVM`, `Random Forest`, `KNN` 等。

3. **模型参数区**（以 SVM 为例）  
   - `C Parameter`： 使用滑块或数值输入框，调整惩罚系数。
   - `Gamma Parameter`： 使用滑块或数值输入框，控制核函数宽度。
   - 草图中两者都以横向 Slider + 右侧实时数值显示的形式呈现。

> 视觉上，这张卡片看起来像一个白色矩形，有上标题区和下参数区，内部元素垂直排列。

### 4.3 模型卡片二：Deep Learning Model（深度学习模型）

位置：右侧卡片，与机器学习模型对称。

组成：

1. **卡片标题与开关**  
   - 标题：`Deep Learning Model`
   - 右上角开关 `ON`。

2. **算法选择**  
   - 标签 `Algorithm`
   - 下拉框默认值：`Convolutional Neural Network (CNN)`。
   - 可以在下拉中加入：`HybridSN`, `3D CNN`, `ResNet` 等。

3. **训练超参数设置**

   - `Epochs`：文本输入框或微调器（默认 50）  
   - `Batch Size`：文本框（默认 32）  
   - `Optimizer`：下拉框（如 `Adam`, `SGD`, `RMSprop`）

> 草图里这三个参数以三行形式呈现，每行左是标签，右是控件（输入框 / 下拉框）。

### 4.4 训练数据划分区：Training Data Split

位于两个模型卡片下方，占整行宽度。

内容：

1. 小标题：`Training Data Split`
2. `Training Data Ratio` 滑块：
   - 左侧标记训练比例，如 `80%`
   - 右侧标记测试/验证比例，如 `20%` 或直接显示 `80% / 42`（草图为了示意）。
   - 用户拖动滑块设置训练集比例。
3. `Random Seed` 文本输入框：
   - 标签：`Random Seed`
   - 右侧输入框填入整数（例如 42），用于保证重复实验可复现。

### 4.5 训练按钮与进度反馈

- 主按钮：`Start Training`，位于数据划分区下方居中。
- 点击后系统行为：
  - 锁定当前模型配置控件，显示训练进度条或日志区域：
    - 例如：`Training SVM Model... 35%`  
    - `Training CNN Model... Epoch 12/50`
  - 进度完成后可以在页面底部显示简要结果提示：`Training Completed. Best OA = 92.5% (CNN).`

## 五、结果与评估模块草图（对应 PDF 第 6–7 页）

当左侧导航选中 **Results & Evaluation** 时，右侧显示“结果与评估”界面。

这个页面是整个系统“最炫”的部分，包含：

1. 分类结果可视化（多图对比）
2. 模型性能指标面板
3. 混淆矩阵与类别精度表

### 5.1 上半部分：分类结果可视化（Classification Result Visualization）

#### 5.1.1 标题区域

- 页眉标题：`Classification Result Visualization`
- 位于页面上方，下面依次排列三个圆形图像。

#### 5.1.2 三个圆形影像视图

三个图像以 **横向排列、圆形裁剪** 的形式展示，如三个圆形卡片：

1. **左图：`Original False Color Image`**
   - 显示原始伪彩色遥感影像：
     - 蓝色海水、浅色沙滩、绿色植被等。
   - 目的是给出“观感上的真实感”。

2. **中图：`Classification Map (CNN Model)` 或具体模型名**  
   - 显示由 CNN 或其他模型输出的分类结果。  
   - 不同地物类别用不同色块填充，颜色通常是高饱和度的纯色（蓝、绿、紫、橙等）。

3. **右图：`Ground Truth Comparison`**
   - 显示“真实标注与分类结果的叠加对比”：  
     - 可能以轮廓线或半透明覆盖的形式，标出分类不一致的区域。
   - 草图中视觉上与中图类似，但色斑有些差异，表示错误区域。

#### 5.1.3 图例（Legend）

三个图像下方是一个横向图例条：

- 每个图例项包含小色块 + 文本标签。示例：
  - 蓝色：`Water Body`
  - 灰色：`Structures`
  - 绿色：`Salt Marsh`
  - 浅紫：`Salt Pans`
  - 黄橙：`Beach/Mudflat`
  - 棕色：`Bareland/Other`

> 这部分在实现时建议做成可勾选的图层开关（点击某个类可高亮/隐藏该类别）。

### 5.2 中部：模型性能指标（Model Performance Metrics）

位于图像区域下方中央，布局分为左右两块：

#### 5.2.1 左侧圆形指标卡片

- **Overall Accuracy** 卡片：
  - 圆形 Gauge 或圆饼样式，中心显示，如 `92.5%`。
  - 下方文字：`Overall Accuracy`.

- **Kappa** 卡片：
  - 类似的圆形可视化，显示 `0.88`。
  - 下方文字：`Kappa`.

这两张卡片等宽并排，视觉上像仪表盘的两个“主指标”。

#### 5.2.2 右侧：Confusion Matrix 区域

- 顶部有一个小标题，例如：`Classification Map (CNN Model)` 或 `Confusion Matrix`.
- 下方是一张 **颜色编码的矩形表格**：
  - 行：真实类别（Ground Truth）
  - 列：预测类别（Predicted）
  - 单元格颜色：  
    - 颜色越深 → 数量越大（正确 / 误分像元多）。
    - 对角线上的格子通常颜色最深（正确分类多）。
  - 单元格内显示数字（例如 95、88、94、96 等）。
- 最右下角可能有某些红色或特殊颜色高亮的格子，表示误分比较严重的类（草图中右下角有一个深色偏红的 92*）。

### 5.3 下半部分：类别精度表（Class Accuracy）

最底部是一张表格区域：

- 小标题：`Class Accuracy`.
- 表头列：
  - `Class`
  - `Producer's Accuracy`
  - `User Accuracy`
- 每行对应一个类别（如 `Water Body`, `Salt Marsh`, `Salt Pans`, `Beach/Mudflat`, `Bareland/Other` 等）。
- 表格中每个精度值可以是百分比或小数（例如 `0.97`, `0.88` 等）。

> 这张表用来补充 confusion matrix 的信息，使评估更完整。

### 5.4 模型选择与切换

在性能指标区域附近，还有一个小的“模型选择”控件：

- 标签：`Select Model`
- 下拉菜单：
  - 选项示例：
    - `Overall Accuracy: SVM`
    - `Convolutional Neural Network (CNN)`
- 当用户切换模型：
  - 上面的 OA/Kappa 数值、混淆矩阵颜色、Class Accuracy 表格都会相应切换。
  - 也可以联动上方的分类图（Classification Map 切换为 SVM 的结果）。

## 六、系统特色总结

1. **模块化设计：**
   - 左侧四个大模块（导入 / 预处理 / 训练 / 结果）清晰分隔工作流。
   - 每个模块页面内部还细分多个卡片，操作步骤非常直观。

2. **强交互的可视化：**
   - 导入阶段即可看到伪彩色图像 + 光谱曲线。
   - 预处理阶段可以在光谱图上“框选波段”。
   - 结果阶段有多视图对比、混淆矩阵、精度表格。

3. **模型对比思路：**
   - 明确区分 Machine Learning Model 与 Deep Learning Model，两张卡片并排展示。
   - 允许同时启用两个模型，并在结果页面进行对比。

4. **评估信息完备：**
   - 整体精度（OA、Kappa）
   - Confusion Matrix（按类别结构化展示）
   - Class-wise PA/UA 精度表  
   -> 足以支撑课程报告 / 论文中的实验章节。

5. **用户友好性：**
   - 所有“开关”都是标识明确的 Toggle，主流程按钮使用蓝色凸显。
   - 滑块、下拉框、表格等构成了典型的数据分析工具界面，非常利于答辩展示与实用拓展。
